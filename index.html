Introdução

O JavaScript é uma linguagem de programação de scripts, orientada a objetos (OOD, sigla em inglês), leve e multi-plataforma. Em um ambiente de hospedagem (um navegador web, por exemplo), o JavaScript pode ser conectado aos objetos de seu ambiente para proporcionar controle programático sobre eles.

Por padrão, o JavaScript contém uma biblioteca de objetos, como Array, Date e Math, e um conjunto central de elementos de linguagem, como operadores, estruturas de controle e declarações. O core do JavaScript pode ser estendido para diversas finalidades, complementando-o com objetos adicionais, como por exemplo:

  • O JavaScript é uma linguagem client-side que amplia a sua linguagem principal, fornecendo objetos para controlar um navegador e seu Document Object Model (DOM). Por exemplo, extensões client-side permitem que uma aplicação insira elementos em um formulário HTML e responda aos eventos do usuário, como cliques do mouse, entrada de formulário e navegação na página.
  • O JavaScript pelo lado do servidor (server-side) estende a linguagem principal, fornecendo objetos relevantes para a execução do JavaScript em um servidor. Por exemplo, as extensões server-side permitem que uma aplicação se comunique com um banco de dados, forneça informações continuamente de uma invocação para outra da aplicação, ou realize manipulações de arquivos em um servidor.

JavaScript e Java

JavaScript e Java são semelhantes em alguns aspectos, mas fundamentalmente diferentes em outros. A linguagem do JavaScript se assemelha a do Java, mas não possui a digitação estática e a strong type checking do Java. O JavaScript segue a maioria da sintaxe de expressão Java, convenções de nomenclatura e construções básicas de controle de fluxo, razão pela qual ele foi renomeado de LiveScript para JavaScript.

Em contraste com o sistema de tempo de compilação de classes Java construído por declarações, o JavaScript suporta um sistema de tempo de execução baseado em um pequeno número de tipos de dados representando valores numéricos, booleanos e string. O JavaScript tem um modelo de objeto baseado em protótipos ao invés do modelo de objeto mais comum baseado em classes. O modelo baseado em protótipos fornece resultados dinâmicos; ou seja, o que é herdado pode variar para objetos individuais. O JavaScript também suporta funções sem nenhuma exigência declarativa especial. As funções podem ser propriedades de objetos, executando como métodos de digitação livre.

Quando comparado com o Java, o JavaScript é uma linguagem muito mais livre. Você não precisa declarar todas as variáveis, classes e métodos. Você não precisa se preocupar se os métodos são públicos, privados ou protegidos, e não precisa implementar interfaces. As variáveis, parâmetros e tipos de retorno de funções não são explicitamente digitados.

function greetMe(yourName) { alert("Hello " + yourName); } greetMe("World");

Variáveis

Usa-se variáveis como nomes simbólicos para valores em uma aplicação. Os nomes das variáveis, chamados de identificadores, devem estar de acordo com algumas regras.

Um identificador JavaScript deve começar com uma letra, underline (_) ou sinal de dólar ($); os caracteres subsequentes também podem ser dígitos (0-9). Como o JavaScript é capaz de identificar as letras maiúsculas e minúsculas, as letras incluem os caracteres "A" até "Z" (maiúsculas) e os caracteres "a" até "z" (minúsculas).

As variáveis aceitam letras ISO 8859-1 ou Unicode, como å e ü em identificadores. Também é possível usar as sequências de escape Unicode como caracteres em identificadores. Alguns exemplos de nomes que podem ser usados são: Number_hits, temp99, e _name.

Declarando variáveis
Pode-se declarar uma variável de três maneiras:

Com a palavra-chave var. Por exemplo:var x = 42.Esta sintaxe pode ser usada para declarar tanto variáveis locais quanto globais.

Simplesmente atribuindo-lhe um valor. Por exemplo:x = 42.Desta maneira sempre declara-se uma variável global. Ela gera um aviso de JavaScript. Não é recomendável usar esta variante.

Com a palavra-chave let. Por exemplo:let y = 13.Esta sintaxe pode ser usada para declarar uma variável de bloco em escopo local. Veja sobre escopo de variáveis abaixo.

Escopo de variáveis

Quando você declara uma variável fora de qualquer função, ela é chamada de variável global, pois está disponível para qualquer outro código no presente documento. Quando você declara uma variável dentro de uma função, ela é chamada de variável local, porque está disponível somente dentro dessa função.

Antes do ECMAScript 2015, o JavaScript não possuía escopo de declaração de bloco; ao contrário, uma variável declarada dentro de um bloco é local para aquela função (ou escopo global) dentro da qual o bloco reside.

Por exemplo, o código a seguir imprimirá 5, porque o escopo de x é a função (ou contexto global) dentro da qual x é declarado, e não o bloco, que neste caso é uma declaração de se.

if (true) { var x = 5; } console.log(x); // 5

Este comportamento muda quando é utilizada a declaração let, introduzida pelo ECMAScript 2015.

if (true) { let y = 5; } console.log(y); // ReferenceError: y is not defined
Variáveis globais

As variáveis globais são, de fato, propriedades do objeto global. Em páginas web o objeto global é janela de visualização, e portanto você pode definir e acessar variáveis globais usando a sintaxe window.variable.

Consequentemente, você pode acessar variáveis globais declaradas em uma janela ou frame a partir de outra janela ou frame, especificando o nome da janela ou frame. Por exemplo, se uma variável chamada phoneNumber for declarada em um documento, você pode se referir a esta variável de um iframe como parent.phoneNumber.

Constantes

Você pode criar uma constante somente para leitura, nomeada constante com a palavra-chave const. A sintaxe de uma constante é a mesma de um identificador variável: deve começar com uma letra, sublinhado ou sinal de cifrão e pode conter caracteres alfabéticos, numéricos ou sublinhados.

const PI = 3.14;

Uma constante não pode mudar de valor através de atribuição ou ser redeclarada enquanto o script estiver em execução. Ela deve ser inicializada com um valor definido.

As regras de escopo das constantes são as mesmas que as do let para bloquear variáveis de escopo. Se a palavra-chave const for omitida, o identificador é assumido como representando uma variável.

Não se pode declarar uma constante com o mesmo nome de uma função ou variável no mesmo escopo. Por exemplo, a constante const const:

// ISTO CAUSARÁ UM ERRO function f() {}; const f = 5; // THIS ISTO TAMBÉM CAUSARÁ UM ERRO function f() { const g = 5; var g; //statements }

Entretanto, os atributos dos objetos não são protegidos, portanto, a seguinte declaração é executada sem problemas:

const MY_OBJECT = {"key": "value"}; MY_OBJECT.key = "otherValue";
Tipos de dados

A última norma ECMAScript define sete tipos de dados:

  • Há seis tipos de dados que são primitivos:
    • Booleano. verdadeiro e falso.
    • null. Uma palavra-chave especial que denota um valor nulo. Como o JavaScript é sensível a maiúsculas e minúsculas, nulo não é o mesmo que Null, NULL ou qualquer outra variante.
    • undefined. Uma propriedade de nível superior cujo valor é indefinido.
    • Número. 42 ou 3.14159.
    • String. "Olá"
    • Símbolo (novidade do ECMAScript 2015). Um tipo de dado cujas instâncias são únicas e imutáveis.
  • Objetos

Embora estes tipos de dados sejam uma quantidade relativamente pequena, eles permitem que você desempenhe funções úteis com suas aplicações. Objetos e funções são os outros elementos fundamentais na linguagem. Você pode pensar em objetos como recipientes nomeados para valores, e funções como procedimentos que sua aplicação pode realizar.

Declaração if... else

Use a declaração if para executar uma declaração se uma condição lógica for verdadeira. Use a outra cláusula opcional para executar uma declaração se a condição for falsa. Uma declaração if possui a seguinte grafia:

if (condition) { statement_1; } else { statement_2; }

condition pode ser qualquer expressão que se avalie como verdadeira ou falsa. Veja Booleano para uma explicação do que se avalia como verdadeiro e falso. Se a condição avalia como verdadeira, a statement_1 é executada; caso contrário, a statement_2 é executada. statement_1 e statement_2 podem ser qualquer expressão, incluindo outras aninhadas se as expressões forem verdadeiras.

Também pode-se compor as expressões usando outras se tiver múltiplas condições testadas em sequência, como a seguir:

if (condition_1) { statement_1; } else if (condition_2) { statement_2; } else if (condition_n) { statement_n; } else { statement_last; }

No caso de múltiplas condições, somente quando a primeira condição lógica é avaliada como verdadeira, a expressão será executada. Para executar declarações múltiplas, deve-se agrupá-las dentro de uma declaração em bloco ({ ... }) . Em geral, é uma boa prática usar sempre as instruções de bloco, especialmente ao aninhar (nest) as instruções:

if (condition) { statement_1_runs_if_condition_is_true; statement_2_runs_if_condition_is_true; } else { statement_3_runs_if_condition_is_false; statement_4_runs_if_condition_is_false; }

É aconselhável não usar atribuições simples em uma expressão condicional, porque a atribuição pode ser confundida com igualdade quando se olha para o código. Por exemplo, não utilize o seguinte código:

if (x = y) { /* statements here */ }

Se você precisar usar uma tarefa em uma expressão condicional, uma prática comum é colocar parênteses adicionais em torno da tarefa. Por exemplo:

if ((x = y)) { /* statements here */ }
Declaração while

Uma declaração while executa suas declarações, desde que uma condição específica se avalie como verdadeira. Uma declaração de while tem o seguinte aspecto:

while (condition) statement

Se a condição se tornar falsa, a declaração dentro do loop interrompe a execução e o controle passa para a declaração que seguinte ao loop.

O teste da condição ocorre antes que a declaração no loop seja executada. Se a condição retornar verdadeira, a instrução é executada e a condição é testada novamente. Se a condição retorna falsa, a execução pára e o controle é passado para a instrução seguinte.

Para executar várias instruções, utilize uma instrução em bloco ({ ... }) para agrupa-las.

Exemplo:

O seguinte loop while é executado enquanto n for menor que 3:

var n = 0; var x = 0; while (n < 3) { n++; x += n; }

A cada iteração, o loop aumenta n e adiciona esse valor a x. Portanto, x e n assumem os seguintes valores:

  • Após o primeiro passo: n = 1 e x = 1
  • Após o segundo passo: n = 2 e x = 3
  • Após o terceiro passo: n = 3 e x = 6

Após completar o terceiro passo, a condição n < 3 não é mais verdadeira, portanto, o loop termina.

Declarações de função

Uma definição de função (também chamada de declaração de função) consiste na palavra-chave de function, seguida por:

  • O nome da função.
  • Uma lista de argumentos para a função, entre parênteses e separados por vírgulas.
  • As declarações JavaScript que definem a função são colocadas entre parênteses curvos, { }.

O código a seguir, por exemplo, define uma função simples chamada square:

function square(number) { return number * number; }

A função square tem um argumento, chamado number. A função consiste em uma declaração que diz para retornar o argumento da função (ou seja, number) multiplicado por si mesmo. A declaração return especifica o valor retornado pela função.

return number * number;

Os parâmetros primitivos (tais como um número) são passados para funções pelo seu valor; o valor é passado para a função, mas se a função muda o valor do parâmetro, esta mudança não é refletida globalmente ou na função de chamada.
